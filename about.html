OPTICAL CALCULATOR: Submission for the BPhO Computational Challenge, 2025

Tony Jiang and William Hood, Winchester College

In 2025, the theme of the BPhO Computational Challenge was optics. This was split into twelve challenges, all of which involve creating computer models to demonstrate various physical occurrences to do with light, and the ability to use mathematical methods to describe them. These ranged from using simple formulae to plot refractive index against frequency/wavelength, to modelling rainbow behaviour.

Although these challenges were more or less solvable using a spreadsheet software like Microsoft Excel, to properly tackle them and provide user-friendly programs with customised GUIs, code was necessary.

At first, given our previous knowledge of Python (from GCSE Computer Science), it seemed that Python would be an ideal language to use. Being a high-level language, coding in Python is simple, and one can easily transfer these python programs into executables, which would allow anyone with a Windows OS to use our GUI. This would mean ease of use for most, with only an .exe file needing to be shared to any potential user. However, there will also be significant drawbacks. Python is a slower language, and also takes up large quantities of memory when doing complex calculations for our needs. Furthermore, it lacks support in mobile environments, meaning only someone with a Windows operating system can easily use it, even when transferred to an .exe file. Download is also required to run these .exe files, which can be of annoyance, especially when we have twelve separate tasks to code.

These propagation and speed problems led us to develop this website. Through integrating our submission into a website, our code can be easily shared through the Internet, and anyone in the world can interact with our programs without download. Our interactive GUI was created and styled with Javascript, HTML, and CSS. In hindsight, JavaScript was ideal for this project. It is executed with JIT compilation, making it faster than Python, while also being native to browsers, which makes its integration into HTML pages extremely straightforward.
