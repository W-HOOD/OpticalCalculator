Optical Calculator: Submission for the BPhO Computational Challenge, 2025 William Hood and Tony Jiang, Winchester College

In 2025, the theme of the BPhO Computational Challenge was optics. This was split into twelve challenges, all of which involve creating computer models to demonstrate various physical occurrences to do with light, and the ability to use mathematical methods to describe them. These ranged from using simple formulae to plot refractive index against frequency/wavelength, to modelling rainbows. Although these challenges were more or less solvable using a spreadsheet software like Microsoft Excel, to properly tackle them and provide user-friendly programs with customised GUIs, code was necessary. At first, given our previous knowledge of Python (from GCSE Computer Science), it seemed that Python would be an ideal language to use. It is a high-level language, meaning it is very easy to code in, especially given our previous experience with it, and one can easily transfer python files into executables, which would allow anyone with a Windows OS to use our programs. This would mean ease of use for most, with only an .exe file needing to be shared to any potential user. There were also, however, significant drawbacks. Python is a slower language, and also takes up large quantities of memory when doing complex calculations (as we need). Furthermore, it lacks support in mobile environments, meaning only someone with a Windows operating system can easily use it, even when transferred to an exe file. One also has to send over a file for it to be accessed, which can be an annoyance, especially if files are large. These propagation and speed problems led us to think of other options. We considered C and C++, however these languages are too low-level and lack the support for visual display that we needed (given the whole program needed its own GUI), and despite its efficiency in performing calculations, still needs a file to be transferred, and a compiler to be installed. In the end we settled for creating a website to contain our models. Not only does this make use of the Internet to make it incredibly easy to share our code with others, but also allows us to make use of javascript to perform the calculations and add interactive GUI elements, HTML to structure the webpages however we like, and CSS to make the whole project look more professional and clean. Javascript, in some ways, is ideal for this project. It is compiled rather than interpreted, meaning that it is faster than python, and also is very easy to integrate into HTML webpages.
